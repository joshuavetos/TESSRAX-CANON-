CONTRIBUTING RULES — TESSRAX

This repository is not open-ended.
Contribution is conditional and adversarial by default.

--------------------------------------------------

GENERAL RULE

Prose does not override code.
Code does not override contracts.
Contracts do not override canon.

--------------------------------------------------

CANON RULES

- Canon is CLOSED unless explicitly declared OPEN
- Canon files are non-derivable
- Changes to canon require:
  • version increment
  • diagnostics proving non-regression
  • explicit canon status update

--------------------------------------------------

ACCEPTABLE CONTRIBUTIONS

- Diagnostics that expose new failure modes
- Tests that prove halting behavior
- Schemas that restrict ambiguity
- Enforcement logic that reduces discretion

--------------------------------------------------

AUTOMATIC REJECTION CONDITIONS

Pull requests will be rejected if they:

- Add discretion
- Add explanation without enforcement
- Add convenience that weakens refusal
- Add narrative framing
- Add features without diagnostics
- Modify enforcement logic without versioning

--------------------------------------------------

BURDEN OF PROOF

The contributor bears the burden of proof.
Claims without artifacts are invalid.

--------------------------------------------------

END CONTRIBUTING RULES
